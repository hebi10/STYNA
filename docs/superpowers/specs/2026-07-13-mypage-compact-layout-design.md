# 마이페이지 컴팩트 레이아웃 설계

## 목표

- 마이페이지 첫 화면과 모든 하위 화면의 과도한 박스 크기와 여백을 줄인다.
- 기존 쇼핑몰의 흑백 디자인과 정보 구조는 유지한다.
- 데스크톱에서 한 화면에 더 많은 정보를 자연스럽게 보여주되 모바일 조작성은 보존한다.

## 화면 구조

### 마이페이지 첫 화면

- 좌측 메뉴 첫 항목으로 `나의 쇼핑 현황`을 추가하고 `/mypage`에서만 활성화한다.
- 회원 요약과 빠른 메뉴는 마이페이지 첫 화면에서만 보여준다.
- 통계 카드는 데스크톱 4열, 태블릿 2열, 모바일 1열을 기본으로 한다.
- 최근 본 상품과 찜한 상품 영역은 기존 기능을 유지하면서 제목, 여백, 버튼을 축소한다.

### 하위 화면

- 주문내역, 주문상세, 문의, 최근 본 상품, 찜한 상품, 쿠폰, 적립금, 회원정보, 상담내역, 재입고알림 화면에서는 회원 요약과 빠른 메뉴를 숨긴다.
- 하위 화면은 공통 페이지 헤더 다음에 좌측 메뉴와 본문이 바로 시작되게 한다.
- 각 메뉴는 자신의 경로에서만 활성화한다. 주문상세는 주문내역 메뉴를 활성화한다.

## 밀도와 스타일

- 데스크톱 좌측 메뉴 폭은 약 220px로 줄이고 메뉴 높이와 내부 여백을 20~30% 축소한다.
- 공통 본문 패딩, 섹션 간격, 카드 패딩을 20~30% 줄인다.
- 큰 제목과 통계 숫자는 쇼핑몰 본문에 맞는 크기로 낮춘다.
- 카드 구분은 얇은 선과 배경 차이로 유지하며 그림자와 새 둥근 모서리를 추가하지 않는다.
- 기존 기능, 데이터 조회, 버튼 동작, 접근성 이름은 변경하지 않는다.
- 모바일에서는 좌측 메뉴를 기존처럼 숨기고 입력 필드와 버튼의 터치 영역을 유지한다.

## 변경 범위

- 공통 레이아웃과 좌측 메뉴
- 마이페이지 대시보드
- 주문 목록과 주문 상세
- 쿠폰과 쿠폰 등록
- 적립금
- 문의 관리
- 회원정보 수정
- 최근 본 상품과 찜한 상품
- 상담내역, 재입고알림 등 공통 레이아웃을 사용하는 하위 화면

## 검증

- `/mypage`에서 `나의 쇼핑 현황`만 활성화되는 회귀 테스트를 추가한다.
- 각 대표 하위 경로에서 올바른 좌측 메뉴가 활성화되는지 확인한다.
- 마이페이지 첫 화면에만 회원 요약과 빠른 메뉴가 표시되는지 확인한다.
- 타입체크, 린트, 전체 테스트, 함수 빌드, Next.js 빌드를 실행한다.
- 데스크톱과 모바일 브라우저에서 첫 화면과 대표 하위 화면을 확인한다.
