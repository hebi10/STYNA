# 이벤트 목록 클린 갤러리 디자인

## 목표

한글 행사명과 혜택 문구가 포함된 캠페인 이미지를 가리지 않고, 실제 종합 패션몰처럼 정돈된 이벤트 목록을 제공한다.

## 화면 구성

- 상단은 개별 이벤트가 아닌 이벤트 허브 전용 고정 이미지를 사용한다.
- 허브 이미지는 링크, 상태, 기간, CTA, 이벤트 현황 오버레이 없이 장식 이미지로 노출한다.
- 허브 이미지는 27:9 비율을 유지하며 데스크톱과 모바일에서 전체가 보이게 표시한다.
- 목록 카드 이미지 위 유형 및 상태 배지도 제거한다.
- 카드 아래에는 이벤트 기간과 변형별 CTA만 남겨 탐색에 필요한 최소 정보를 제공한다.
- 행사명과 혜택은 이미지 안의 한글 문구로 전달하며 UI에서 반복하지 않는다.

## 이벤트 허브 이미지

- 한 장의 상시용 에디토리얼 이미지로 제작한다.
- 이미지 안 문구는 `STYNA EVENTS`, `새로운 스타일과 혜택을 만나보세요` 두 개만 사용한다.
- 여성·남성 혼성 모델과 의류·가방·신발 제품 디테일을 조합한다.
- 특정 계절, 특정 행사명, 할인율, 쿠폰 숫자는 넣지 않는다.
- 블랙, 아이보리, 차콜 기반에 코발트 블루 포인트를 사용한다.

## 그리드와 이미지 비율

- 데스크톱 목록은 정확히 4열로 표시한다.
- 태블릿은 2열, 모바일은 1열로 전환한다.
- 카드 목록은 페이지당 정확히 8개를 노출해 데스크톱에서 4열 × 2줄을 구성한다.
- 카드 이미지는 제작 원본과 같은 `4 / 5` 비율을 사용한다.
- 이미지는 전체가 보이도록 `object-fit: contain`을 적용하고 확대 효과를 제거한다.
- 이미지와 카드에는 새 그림자나 라운드를 추가하지 않는다.

## 접근성 및 동작 보존

- 대표 허브 이미지는 링크를 제공하지 않고, 각 카드만 기존 상세 페이지 링크를 유지한다.
- 화면 텍스트를 줄여도 링크의 접근 가능한 이름은 이벤트 제목과 설명을 유지한다.
- 필터, 페이지네이션, 로딩·오류·빈 상태 동작은 변경하지 않는다.

## 검증

- 컴포넌트 테스트에서 대표 및 카드 오버레이가 렌더링되지 않는지 확인한다.
- 이벤트 provider 테스트에서 페이지당 노출 수가 8인지 확인한다.
- 컴포넌트 테스트에서 허브 이미지가 링크로 감싸지지 않고 카드가 최대 8개만 렌더링되는지 확인한다.
- CSS 정적 테스트에서 데스크톱 4열, 카드 `4 / 5`, `object-fit: contain`을 확인한다.
- 관련 테스트, 타입 검사, 린트 후 브라우저에서 대표 이미지와 4열 카드의 잘림·가로 넘침을 확인한다.
