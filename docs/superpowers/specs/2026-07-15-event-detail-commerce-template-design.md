# 이벤트 상세 커머스 템플릿 리뉴얼 설계

## 배경

이벤트 상세 화면은 현재 제목, 기간, 참여자, CTA, 캠페인 이미지, 혜택 카드, 본문, 참여 방법, 유의사항, 요약을 각각 큰 박스로 반복한다. `라스트 썸머 클리어런스` 모바일 화면은 약 4,391px까지 길어지고, 행사명과 할인 혜택이 이미지와 UI에서 여러 번 반복된다. 결과적으로 실제 쇼핑몰 기획전보다 운영 안내 페이지처럼 보이며 상품 탐색으로 이어지는 흐름이 약하다.

## 목표

- Firestore 이벤트 22개가 하나의 공통 상세 템플릿을 사용한다.
- 세일, 쿠폰, 리뷰, 신상품, 특별 이벤트는 같은 구조 안에서 카피와 행동, 추천 상품 소스만 달라진다.
- 캠페인 이미지와 실제 상품 탐색을 중심으로 페이지를 재배치한다.
- 기존 참여, 쿠폰 지급, 로그인, 상태 판정, HTML 정화 로직은 보존한다.
- 모바일에서도 핵심 행동과 혜택을 짧은 이동 거리 안에서 확인할 수 있게 한다.

## 비목표

- Firestore 이벤트 스키마를 새로 만들거나 기존 22개 문서를 다시 마이그레이션하지 않는다.
- 이미지 44개를 다시 생성하거나 Storage 경로를 변경하지 않는다.
- 상품 가격, 재고, 쿠폰 발급 정책, 이벤트 참여 API를 변경하지 않는다.
- 이벤트마다 별도의 수작업 페이지를 만들지 않는다.

## 선택한 방향

커머스 중심 공통 템플릿을 사용한다. 캠페인 이미지 다음에 핵심 혜택과 CTA를 배치하고, 관련 상품을 최대 8개 노출한 뒤 상세 안내와 유의사항을 압축해서 제공한다. 긴 에디토리얼형이나 정보만 남기는 축약형보다 현재 종합 패션몰의 데이터와 구매 흐름에 가장 적합하다.

## 화면 구조

### 1. 상단 내비게이션

- 기존 헤더와 breadcrumb를 유지한다.
- breadcrumb 아래의 넓은 빈 공간과 별도 대형 텍스트 히어로는 제거한다.

### 2. 캠페인 히어로

- 데스크톱은 `detailImage` 와이드 이미지, 640px 이하는 `thumbnailImage` 카드 이미지를 사용한다.
- 이미지 안에 행사명과 혜택 문구가 있으므로 제목·할인 문구를 이미지 위에 다시 오버레이하지 않는다.
- 이미지 바로 아래에 작은 상태 배지, 의미론적 `h1`, 한 줄 설명, 기간을 배치한다.
- 참여자 수는 `hasMaxParticipants`가 참이고 실제 정원이 있을 때만 `현재 / 정원` 형태로 표시한다.

### 3. 핵심 혜택·행동 바

- 혜택, 대상 범위, 기간을 한 줄 요약으로 배치한다.
- 주 CTA는 이벤트 상태와 타입에 따라 기존 행동을 그대로 사용한다.
- 모바일에서는 같은 CTA를 화면 하단 고정 바에 노출하고 본문에는 고정 바 높이만큼 여백을 둔다.
- 모바일 고정 바는 이벤트 상세 경로에서만 렌더링하고 기존 쇼핑 안내·상담 플로팅 UI와 겹치지 않는지 390px 화면에서 확인한다. 다른 경로의 플로팅 UI 동작은 변경하지 않는다.
- 예정 이벤트는 오픈 일정 확인, 종료 이벤트는 다른 이벤트 또는 관련 상품 보기로 전환한다.

### 4. 관련 상품 쇼케이스

- 기존 `ProductCard`를 재사용해 데스크톱 4열, 모바일 2열로 최대 8개를 노출한다.
- 선택 우선순위는 다음과 같다.
  1. `targetProducts`가 있으면 병렬 조회하되 원래 ID 순서를 복원하고, 존재하며 `status === 'active'`인 상품만 사용한다.
  2. `targetCategories`가 있고 `전체`가 아니면 카테고리별 상품을 조회해 중복을 제거한다.
  3. 명시 대상이 없거나 결과가 부족하면 이벤트 변형에 맞는 상품으로 채운다.
- 변형별 기본 상품 소스:
  - 세일: `ProductService.getSaleProducts()`
  - 쿠폰: `ProductService.getRecommendedProducts()`
  - 리뷰: `ProductService.getReviewPopularProducts()`
  - 신상품: `ProductService.getNewProducts()`
  - 특별: `ProductService.getRecommendedProducts()`
- 상품 로딩 실패는 이벤트 페이지 전체 오류로 전파하지 않는다. 쇼케이스 안에서 재시도 버튼과 상품 전체 보기 링크를 제공한다.
- 노출 상품이 없으면 빈 카드 대신 이벤트 타입에 맞는 상품 목록 CTA만 보여준다.

### 5. 상세 정보

- 현재 반복되는 4개 혜택 카드, 소개 박스, 4개 정보 패널을 다음 세 영역으로 합친다.
  - 혜택 안내
  - 참여·사용 방법
  - 유의사항
- 데스크톱은 세 영역을 구분선 기반의 2열/단일 열 조합으로 보여준다.
- 모바일은 네이티브 버튼 기반 접기/펼치기 구조를 사용하며 첫 번째 영역만 기본으로 연다.
- `event.content`는 기존 `sanitizeEventHtml()`을 거쳐 혜택 안내 안에서 렌더링한다.
- 같은 기간, 대상 범위, 지급 방식을 여러 패널에서 반복하지 않는다.

### 6. 하단 전환 영역

- 큰 보조 행동 박스는 제거한다.
- `관련 상품 더 보기`와 `전체 이벤트` 두 개의 텍스트/검정 CTA만 남긴다.
- 종료 이벤트는 진행 중 이벤트로 이동하는 CTA를 우선한다.

## 이벤트 타입별 변형

| 변형 | 핵심 혜택 | 주 CTA | 상품 섹션 제목 | 강조 방식 |
| --- | --- | --- | --- | --- |
| 세일 | 할인율 또는 할인 금액 | 할인 상품 보기 | 지금 할인 중인 상품 | 버건디 얇은 선 |
| 쿠폰 | 쿠폰 금액·코드·지급 방식 | 쿠폰 받기 | 쿠폰과 함께 보기 좋은 상품 | 블루 얇은 선 |
| 리뷰 | 리뷰 보상 | 리뷰 쓰고 참여하기 | 리뷰가 많은 상품 | 그린 얇은 선 |
| 신상품 | 오픈 혜택 | 신상품 보기 | 새로 들어온 상품 | 퍼플 얇은 선 |
| 특별 | 이벤트별 보상 | 이벤트 참여하기 | 함께 보면 좋은 상품 | 검정 얇은 선 |

색상은 배지와 2px 강조선에만 사용한다. 그림자, 그라데이션, 큰 라운드는 추가하지 않는다.

## 컴포넌트 구조

- `EventDetailClient`
  - 기존 상태 판정, 로그인, 참여 요청, CTA 라우팅을 소유한다.
  - 화면용 데이터와 콜백을 하위 컴포넌트에 전달한다.
- `EventCommerceHero`
  - 반응형 캠페인 이미지와 제목·기간·상태 요약을 렌더링한다.
- `EventActionBar`
  - 혜택 요약, 상태별 CTA, 피드백을 렌더링한다.
- `EventProductShowcase`
  - 상품 소스 선택, 로딩, 오류, 빈 상태와 `ProductCard` 그리드를 담당한다.
- `EventInformationSections`
  - 정화된 본문, 참여 방법, 혜택, 유의사항을 중복 없이 묶는다.
- `EventMobileStickyAction`
  - 640px 이하에서 주 CTA와 상태 문구를 고정한다.

기존 `EventDetailClient`의 순수 계산 함수는 가능한 한 별도 helper로 이동해 컴포넌트가 화면 조합과 행동 연결에 집중하게 한다. 요청과 무관한 서비스 리팩터링은 하지 않는다.

## 데이터 흐름

1. 기존 상세 page가 Firestore 이벤트를 조회해 `EventDetailClient`에 전달한다.
2. `getEventUiVariant()`와 기존 helper가 상태, 혜택, CTA, 참여 방법을 계산한다.
3. 히어로와 정보 컴포넌트는 계산된 view model을 렌더링한다.
4. 상품 쇼케이스가 명시 상품, 대상 카테고리, 타입별 기본 상품 순으로 최대 8개를 비동기 조회한다.
5. CTA는 기존 참여 API, 쿠폰함, 세일, 추천, 리뷰, 이벤트 목록 라우팅을 그대로 사용한다.

## 오류·상태 처리

- 이벤트 조회 실패와 존재하지 않는 이벤트 처리는 기존 상세 page 동작을 유지한다.
- 상품 로딩 실패는 인라인 오류와 재시도로 격리한다.
- 참여 처리 중에는 본문 CTA와 모바일 고정 CTA를 동시에 비활성화한다.
- 참여 성공·실패 피드백은 두 CTA가 공유하는 하나의 `aria-live` 영역에서 제공한다.
- 이미지 fallback과 데스크톱/모바일 소스 선택은 기존 `getEventDisplayImages()`와 `EventResponsiveImage`를 사용한다.
- 긴 행사명, 쿠폰 코드, 날짜 문구는 줄바꿈되며 가로 스크롤을 만들지 않는다.

## 테스트 계획

- 순수 helper 테스트
  - 타입별 상품 소스 선택
  - 명시 상품·카테고리 우선순위와 중복 제거
  - 참여자 수 조건부 표시
  - 예정·진행·종료 CTA 분기
- 컴포넌트 테스트
  - 이미지 위 중복 행사명·혜택 오버레이가 없음
  - 세일·쿠폰·리뷰·신상품·특별 변형별 CTA와 상품 제목
  - 상품 로딩·오류·빈 상태
  - 모바일 고정 CTA와 동일한 비활성 상태
  - 정화된 HTML 렌더링 유지
- 브라우저 QA
  - 데스크톱 1440px와 모바일 390px
  - 세일·쿠폰·리뷰·신상품·특별 대표 이벤트 각 1개
  - 이미지 로딩, CTA, 상품 카드, 접기/펼치기, 가로 오버플로우, 콘솔 오류

## 완료 기준

- Firestore 이벤트 22개가 새 공통 템플릿에서 정상 렌더링된다.
- 첫 화면에서 캠페인 이미지, 핵심 혜택, 기간, 주 CTA가 확인된다.
- 진행 중인 이벤트는 1440×1000과 390×844 화면 모두에서 첫 화면 아래 두 뷰포트 높이 안에 관련 상품 섹션이 시작된다.
- 참여자 0명과 제한 없음이 일반 이벤트 첫 화면에 노출되지 않는다.
- 현재의 반복 제목·기간·혜택 박스가 제거된다.
- 모바일 이벤트 상세 높이와 정보 반복이 유의미하게 줄고 가로 오버플로우가 없다.
- 기존 참여, 로그인, 쿠폰, 리뷰, 상품 이동 동작이 회귀하지 않는다.
- 타입체크, 린트, 관련 테스트, 전체 빌드와 브라우저 QA가 통과한다.
