# 주문·권한 정합성 설계

## 목표

주문 재고, 쿠폰 발급, QnA 수정, 이벤트 참여를 신뢰할 수 있는 서버 트랜잭션과 최소 권한 규칙으로 재구성한다.

## 적용 범위

- 같은 상품의 여러 옵션 주문은 상품별 총수량으로 재고를 한 번만 차감·복원한다.
- 주문 서버는 판매 상태와 상품이 선언한 옵션을 검증한다.
- 일반 사용자의 `issue` 쿠폰 자기 발급을 중단한다. 쿠폰은 코드 등록, 관리자 할당, 이벤트 보상만으로 발급한다.
- 비밀 QnA는 작성자 또는 관리자만 읽는다. 비밀번호 공유 검증 API는 제거해 4자리 PIN 대입 공격을 없앤다.
- QnA 작성자는 본문 편집 필드만 변경할 수 있고 답변·상태·작성자·조회수는 변경하지 못한다.
- 이벤트 참여와 자동 쿠폰 보상은 Function의 단일 Firestore transaction에서 처리한다.
- Firestore 규칙은 Emulator 기반 테스트로 검증한다.

## 비범위

- 실제 PG 결제, 배송사 연동, 회원 배송지 CRUD는 다음 구매 여정 단계에서 처리한다.
- 기존 사용자의 메인 배너 작업 파일은 수정하지 않는다.

## 설계 결정

### 주문 재고

주문 문서의 상품 행은 옵션별로 보존한다. 다만 재고 검증·차감·취소 복원에는 `productId`별 집계 수량만 사용한다. 상품에 `sizes` 또는 `colors` 배열이 있으면 요청 옵션을 해당 배열에서 검증하고, 상태가 명시된 상품은 `active`만 주문을 허용한다.

### 쿠폰과 이벤트

`issue` 액션은 관리자만 사용할 수 있는 관리 기능으로 제한한다. 이벤트는 `POST /api/event/participate`에 `eventId`만 보내며 UID, 참여자 수, 보상 쿠폰 ID는 서버가 결정한다. Function은 결정적 참여 문서 ID(`{eventId}_{uid}`)와 하나의 transaction으로 중복 참여, 정원 초과, 쿠폰 중복 발급을 막는다.

### QnA

QnA 작성은 로그인 필수이므로 비밀글의 비밀번호 공유 조회를 지원하지 않는다. 작성자·관리자 이외의 비밀글 접근은 거부한다. 작성자는 지정된 편집 필드만 업데이트할 수 있으며 관리자가 답변과 상태를 관리한다.

### 검증

순수 도메인 테스트는 재고 집계와 주문 가능 옵션을 검증한다. Function 핸들러 테스트는 transaction 쓰기 단위를 검증한다. Firestore Emulator 테스트는 anonymous/user/owner/admin 매트릭스로 실제 규칙을 검증한다.

## 성공 기준

- 동일 상품의 서로 다른 옵션을 함께 주문·취소해도 재고 변경은 상품당 한 번이며 합계와 일치한다.
- draft/inactive 상품과 선언되지 않은 옵션 주문이 서버에서 거부된다.
- 일반 사용자의 쿠폰 `issue`가 거부되고, 이벤트 보상은 중복 없이 원자적으로 발급된다.
- QnA 작성자가 답변·상태·조회수·작성자 필드를 바꾸지 못한다.
- 일반 사용자의 이벤트·참여 문서 쓰기가 Firestore 규칙에서 거부된다.
- 새 테스트와 기존 품질 게이트가 통과한다.
